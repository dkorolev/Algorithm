This one's from the Google Code Jam. I know, they have their own board. They've even discussed this problem (http://groups.google.com/group/Code-Jam-2006/browse_thread/thread/5cb1999141950ac6/#). I still don't understand, though; I thought I might get more helpful assistance here.

So given an N x N matrix (N <= 16), the problem asks us to find the permanent of the matrix. I've been trying to understand Kawigi's solution; it seemed pretty elegant. Here's the code, with (hopefully correct) comments added because I'm not used to reading bitwise operations:

public int compute(String[] matrix) {
	int[][] m = splitInts2D(matrix);
	int[] p = new int[1 << m.length];
	p[0] = 1;
 
	for (int i = 1; i < p.length; i++) {
		//r = n - (the number of 1 bits in i)
		int r = m.length;
		for (int j = 0; j < m.length; j++)
			r -= (i >> j) & 1;
 
		for (int j = 0; j < m.length; j++)
		{
			//if i has a 1 bit at location j
			if ((i & (1 << j)) != 0)
			{
				//i ^ (1 << j) = i with the bit at j turned off
				int ind = i ^ (1 << j);
				p[i] = (p[i] + m[r][j] * p[ind]) % 10000;
			}
		}
	}
 
	return p[p.length - 1];
}


This seems just beyond my grasp. I know we're memoizing submatricies, but I don't understand the pattern. I guess what's most confusing to me is: How does all this bit twiddling give us full coverage of the permutations we want to consider?

Thanks for whatever light you can shed on this problem.

aussie
3270 posts
Your comments look correct. To quote Kawigi from another thread where this problem was discussed: "Given some bitmask on the columns, there's only one interesting submatrix that uses those columns, and the rows it includes are the last n rows where n is the number of bits set on the bitmask."

Does that help you understand his code a bit better?

The first part you commented where he calculates r is to find the top row of the current submatrix. Then the next loop iterates over all the columns, checks the bitmask to determine if each column is in the current submatrix and if it is it then adds the previously calculated permanent of the submatrix with that row and column removed to the result for the current submatrix.

Same as Larry said in other words:

You need to represent a matrix containing last K rows and some K columns. This state can be described using one int (the value K) and one bitset (which K columns).

As a bonus, now you can realize that using the value K is not necessary, as K = number of ones in the bitset. (But AFAIK this step wasn't even necessary.)
